Sessions & Cookies Concepts

Session Storage

Stores data for the duration of the page session. Data is cleared when tab is closed.

Local Storage

Persists data indefinitely until explicitly cleared. Survives browser restarts.

Cookies

Small files stored on client side. Sent with every HTTP request. Useful for authentication.

Session ID

Unique identifier for each user session. Validates user identity across pages.

Expiry Management

Automatic deletion of sessions/cookies after specified time period for security.

Security

Use secure flags, httponly attributes, and proper validation to protect user data.

Session Variables vs Cookies:

Feature Session Cookies
Storage Location Server (or client storage) Client (browser)
Lifetime Until browser closes Until expiration date
Sent with Requests No (server-side only) Yes (HTTP header)
Security More secure Less secure (visible)
Size Limit Unlimited (server) 4KB per cookie
Use Case Sensitive data, authentication Preferences, tracking
